# Security Policy

## Supported branch

La única rama soportada para correcciones de seguridad es `main`.

## Private reporting

No abras issues públicos para reportar vulnerabilidades. Utiliza
[GitHub Private Vulnerability Reporting](https://github.com/j-a-a-s/kaklen/security/advisories/new).

El reporte no debe incluir secretos, tokens, datos personales ni exploits en
canales públicos. Incluye en el advisory privado la información mínima necesaria
para reproducir y evaluar el riesgo de forma segura.

`j-a-a-s` coordina la validación, priorización, corrección y divulgación de cada
reporte confirmado.
